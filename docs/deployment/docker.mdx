# Docker Deployment

Deploy Cally using Docker Compose for a containerized setup with PostgreSQL, Redis, the API server, the web frontend, and the background worker.

## Docker Compose Configuration

Create a `docker-compose.yml` in your project root:

```yaml
version: "3.8"

services:
  postgres:
    image: postgres:16-alpine
    restart: always
    environment:
      POSTGRES_USER: cally
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-changeme}
      POSTGRES_DB: cally
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U cally"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    restart: always
    command: redis-server --requirepass ${REDIS_PASSWORD:-changeme}
    volumes:
      - redis_data:/data
    ports:
      - "6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  api:
    build:
      context: .
      dockerfile: Dockerfile.api
    restart: always
    environment:
      NODE_ENV: production
      HOST: 0.0.0.0
      PORT: 4000
      DATABASE_URL: postgresql://cally:${POSTGRES_PASSWORD:-changeme}@postgres:5432/cally
      BETTER_AUTH_SECRET: ${BETTER_AUTH_SECRET}
      BETTER_AUTH_URL: ${BETTER_AUTH_URL:-http://localhost:4000}
      CLIENT_URL: ${CLIENT_URL:-http://localhost:3000}
      CORS_ORIGINS: ${CORS_ORIGINS:-http://localhost:3000}
      REDIS_URL: redis://:${REDIS_PASSWORD:-changeme}@redis:6379
      MASTER_ENCRYPTION_KEY: ${MASTER_ENCRYPTION_KEY}
    ports:
      - "4000:4000"
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy

  web:
    build:
      context: .
      dockerfile: Dockerfile
    restart: always
    environment:
      NEXT_PUBLIC_BACKEND_URL: ${NEXT_PUBLIC_BACKEND_URL:-http://localhost:4000}
      NEXT_PUBLIC_APP_NAME: ${APP_NAME:-Cally}
      NEXT_PUBLIC_APP_URL: ${APP_URL:-http://localhost:3000}
    ports:
      - "3000:3000"
    depends_on:
      - api

  worker:
    build:
      context: .
      dockerfile: Dockerfile.api
    restart: always
    command: ["node", "dist/workers/index.js"]
    environment:
      NODE_ENV: production
      DATABASE_URL: postgresql://cally:${POSTGRES_PASSWORD:-changeme}@postgres:5432/cally
      BETTER_AUTH_SECRET: ${BETTER_AUTH_SECRET}
      BETTER_AUTH_URL: ${BETTER_AUTH_URL:-http://localhost:4000}
      REDIS_URL: redis://:${REDIS_PASSWORD:-changeme}@redis:6379
      MASTER_ENCRYPTION_KEY: ${MASTER_ENCRYPTION_KEY}
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy

volumes:
  postgres_data:
  redis_data:
```

## Environment File

Create a `.env` file in the same directory:

```env
# Generate secrets with: openssl rand -base64 32
POSTGRES_PASSWORD=your_secure_postgres_password
REDIS_PASSWORD=your_secure_redis_password
BETTER_AUTH_SECRET=your_long_random_auth_secret_at_least_16_chars
MASTER_ENCRYPTION_KEY=your_base64_encryption_key

# URLs
BETTER_AUTH_URL=https://your-domain.com
CLIENT_URL=https://your-domain.com
CORS_ORIGINS=https://your-domain.com
NEXT_PUBLIC_BACKEND_URL=https://your-domain.com
APP_NAME=Cally
APP_URL=https://your-domain.com
```

## Build and Run

```bash
# Build images
docker compose build

# Start all services
docker compose up -d

# Run database migrations
docker compose exec api npx prisma migrate deploy

# Check logs
docker compose logs -f api
docker compose logs -f worker

# Stop all services
docker compose down

# Stop and remove volumes (⚠️ deletes data)
docker compose down -v
```

## Production Considerations

### Reverse Proxy

Place a reverse proxy (Caddy, Nginx, or Traefik) in front of the containers:

```caddy
your-domain.com {
    reverse_proxy web:3000
    handle /api/* {
        reverse_proxy api:4000
    }
    handle /health {
        reverse_proxy api:4000
    }
}
```

### Database Backups

```bash
# Backup
docker compose exec postgres pg_dump -U cally cally > backup_$(date +%Y%m%d).sql

# Restore
cat backup_20260524.sql | docker compose exec -T postgres psql -U cally cally
```

### Scaling

To scale the worker for more concurrent sync jobs:

```bash
docker compose up -d --scale worker=3
```
