## Module Architecture

Each domain module in the backend follows a consistent layered pattern:

```
modules/<domain>/
├── routes.ts       # Fastify route definitions + validation
├── schemas.ts      # Zod request/response schemas
├── service.ts       # Business logic
├── repository.ts    # Database queries (Prisma)
└── types.ts         # TypeScript type definitions
```

This separation ensures:
- **Routes** handle HTTP concerns (headers, status codes, cookies)
- **Services** contain pure business logic
- **Repositories** encapsulate database access
- **Schemas** define validated data shapes using Zod
- **Types** provide TypeScript interfaces for the domain

## Package Dependencies

```
@cally/client  ──depends-on──>  @cally/contracts
@cally/backend ──depends-on──>  @cally/contracts
```

The `@cally/contracts` package is the **single source of truth** for API request/response shapes. Both the client and backend import Zod schemas from this package to ensure type-safe, runtime-validated data transfer.
