# Frontend Overview

The Cally frontend is a **Next.js 16 App Router** application using React 19, TypeScript, and Tailwind CSS v4. It communicates exclusively with the standalone Fastify backend via the `apiJson` helper.

## Authentication Flow

The frontend uses `better-auth/react` client with a NextAuth-compatible shim:

```typescript
// src/lib/auth-react-compat.tsx provides:
// - useSession() → { data, status, error, update }
// - signIn(provider, options) → supports "credentials" + OAuth
// - signOut(options) → clears session + redirects
// - SessionProvider → wraps the app
```

Public pages (`/login`, `/register`, `/book/*`, `/maintenance`) render without the navigation shell. All other pages check session status and redirect to `/login` if unauthenticated.

## API Client

All backend calls go through the `apiJson` helper:

```typescript
// src/lib/api-client.ts
const backendUrl = process.env.NEXT_PUBLIC_BACKEND_URL ?? "http://localhost:4000";

export async function apiJson<T>(path: string, options?: RequestInit): Promise<T> {
  const res = await fetch(`${backendUrl}${path}`, {
    ...options,
    headers: {
      "Content-Type": "application/json",
      ...options?.headers,
    },
    credentials: "include", // Include cookies for auth
  });

  if (!res.ok) {
    const error = await res.json();
    throw new Error(error.message || "API request failed");
  }

  return res.json();
}
```

## Theming

The app uses CSS custom properties for theming, defined in `globals.css`:

- **Light/dark mode** via the `ThemeProvider` component (persists to `localStorage`)
- **Branding** via `BrandingProvider` which fetches `/api/v1/branding` and applies CSS variables
- **Tailwind v4** with custom color tokens mapped to CSS variables
- An inline script in `layout.tsx` reads `cally-theme` from localStorage to prevent flash of wrong theme

## Onboarding

New users see a 5-step wizard:

1. **Welcome** — Intro message
2. **Timezone** — Set timezone and country
3. **Availability** — Configure weekly hours
4. **Event Type** — Create first appointment type
5. **Done** — Redirect to dashboard

The wizard sets `onboardingCompleted = true` on the user profile after completion.
